## Round 1 - Technical Correctness
- Stage2 ROI 설명에서 `rois`의 첫 열(`batch_index`) 의미를 코드와 맞게 명확화했다. `feat[b_idx:b_idx+1]`를 쓰는 흐름에서는 값이 0으로 고정됨을 추가했다.
- Stage2 head 디코딩 설명을 `전역 좌표`에서 `정규화 전역(cxcywh) 좌표`로 명확히 바꿔 좌표 단위 혼동을 줄였다.
- 12장 제목/학습목표/본문에서 절대 좌표 표현을 `0~1 정규화 절대 좌표`로 통일해 coordinate convention을 분명히 했다.
- 12장 코드 지도 문구를 `image-global 정규화(cxcywh)`로 수정해 실제 출력 타입(`pred_head_boxes`)과 일치시켰다.
- 14장 wrapper 예시 코드의 `body_head_set_loss` 기본 `w_head` 값을 20.0에서 4.0으로 수정해 실제 함수 시그니처와 맞췄다.
- 14장 wrapper 예시에서 `cascaded_body_head_loss`가 기본 가중치를 다르게 넘길 수 있는 형태로 바꿔 실제 동작(재사용 + 기본값 차이)을 정확히 반영했다.
- 17장 손실 조합 설명에 `normalize_by="matched"`일 때 IoU 항이 `2*matched`로 정규화된다는 점을 추가해 손실 스케일 설명을 코드와 맞췄다.

## Round 2 - Pedagogy
- 2장, 3장, 4장, 5장, 11장, 16~20장 시작부에 `이전 장 -> 이번 장` 연결 문장을 넣어 장 간 이동 흐름을 또렷하게 만들었다.
- 5장 끝의 파트 예고 문장을 6~10장 실제 내용(objectness -> body -> sigmoid -> ROI Align -> 좌표 변환)과 맞게 교정해 독자 혼선을 줄였다.
- 3장 좌표계 설명에서 "단위 태그 먼저 쓰기" 원칙으로 문장을 짧게 재구성해 정규화/피처/원본 좌표 혼동을 줄였다.
- 4장과 5장의 해상도/백본 비교 문단을 짧은 문장으로 분해해 "연산량 절약 vs 작은 물체 단서 손실" 트레이드오프를 더 쉽게 읽게 만들었다.
- Part 4-2 도입부를 "출력 이해 -> loss 학습" 흐름으로 다시 연결해 15장 이후 학습 파트 진입이 자연스럽게 이어지도록 정리했다.
- 16장에 `오해/바로잡기`를 추가해 `pos_weight` 과튜닝과 objectness-박스 품질 혼동을 분리해 설명했다.
- 17장에 `오해/바로잡기`를 추가해 SmoothL1/IoU의 역할 겹침 오해를 바로잡고, 초반 학습 안정성 맥락을 보강했다.
- 18장에 `오해/바로잡기`를 추가해 "큰 가중치 하나만 키우기" 오해를 교정하고, 기여도(가중치*평균손실) 기반 판단을 강조했다.
- 19장과 20장에 `오해/바로잡기`를 추가해 "loss 감소=정상 학습" 오판, "`kept_slots=0` 즉시 실패" 오판을 예방하도록 디버깅 순서를 강화했다.

## Round 3 - Math
- 본문의 모든 블록 수식(15개)을 다시 점검하고, 각 수식 바로 아래에 1문장 해석을 추가해 식-의미 연결을 강화했다.
- 1장 head decode 식을 코드와 동일하게 `head_cx = (body_cx - 0.5*body_w) + rel_cx*body_w` 형태로 정리하고, `clamp(0,1)` 후처리까지 설명했다.
- 2장 objectness 수식을 `p=sigmoid(logit)` 기반의 직관형 BCE 표현으로 바꿔 로짓/확률 관계를 더 쉽게 읽도록 수정했다.
- 3장/10장 `cxcywh <-> xyxy` 변환식은 코드(`cxcywh_to_xyxy`)와 일치하도록 유지하고, "중심에서 반폭/반높이 이동" 해석 문장을 추가했다.
- 6장 MLP 로짓 식, 7장 reshape 식, 9장 ROIAlign 식에서 과한 기호(`R^n`, hat 표기 등)를 줄이고 텐서 shape 중심으로 단순화했다.
- 8장 sigmoid 식에 `z=0 -> 0.5` 해석을 붙여 포화/범위 직관을 강화했다.
- 16장 `pos_weight` 식을 코드 연산과 같은 형태(양성 항 가중)로 정리해 BCEWithLogits 설명과 충돌이 없도록 맞췄다.
- 17장 손실 식을 `SmoothL1(body/head) + (1-IoU_body) + (1-IoU_head)`로 재작성해 실제 코드의 body/head 분리 계산을 반영했다.
- 18장 총손실 가중합, 19장 `matched_ratio`, 20장 train->infer->interpret 루프 식에 각각 짧은 해석 문장을 추가해 실험 판단 문맥을 명확히 했다.

- `from experiments...`를 사용하는 실습 스니펫(1, 2, 3, 5, 20장)에 `Path.cwd()` 기준 상위 디렉터리에서 `experiments/`를 찾는 import preamble을 추가해, repo 내부 임의 위치 실행 시 import 실패 가능성을 줄였다.
- 1장 실험 안내 문구를 "repo root 고정 가정"에서 "자동 경로 보정 방식"으로 바꿔 실행 전제 조건을 코드와 일치시켰다.
- 5장 백본 비교 스니펫에서 변수명을 `bb/m` -> `backbone_name/detector`로 정리해 의미가 즉시 드러나도록 개선했다.
- 15장 Greedy vs 전역최적 비교 스니펫의 하드코딩된 `3x3` 인덱싱을 `row_count/col_count` 기반으로 수정하고, 비정사각 행렬 입력 시 명시적으로 예외를 내도록 바꿔 shape 의존 버그를 줄였다.
- 20장 실험 1에서 loss 출력 변환을 `to_printable` 헬퍼로 변경해 스칼라/비스칼라 tensor 혼합 출력 시 `.item()` 관련 런타임 리스크를 줄였다.
- 20장 실험 2에서 `out`, `d`를 `outputs`, `loss_details`로 바꿔 학습 루프에서 텐서 의미를 명확히 했다.
- 20장 실험 3에서 `pred`/`num` 등 축약 변수를 `first_image_pred`/`num_kept`로 명확화하고, 면적 계산에 `clamp(min=0)`를 넣어 음수 면적 출력 가능성을 방어했다.
- 장별 코드블록 수를 재검증해 20개 모든 장이 최소 3개 이상의 실습 코드(````python`)를 유지함을 확인했다.

## Round 5 - Structure
- Part/챕터 제목 스타일을 전 구간에서 일관 포맷으로 통일했다.
- 20개 챕터 모두 동일한 필수 섹션 순서(학습 목표 -> 이야기로 시작 -> 코드 산책 -> 레이어 왜 쓰나 -> 작은 실험 -> 한 줄 수학 -> 체크 퀴즈 -> 미니 미션)를 적용했다.
- 챕터 간 중복되거나 충돌 가능성이 있는 문장을 제거하고 섹션 단위 문장을 간결화했다.
- 모든 챕터 끝에 1줄 전환 문장을 보장해 흐름 연결을 맞췄다.
- Part 경계(1~5, 6~10, 11~15, 16~20) 헤딩을 동일 스타일로 맞췄다.
- 구조 정합성 중심으로 최소 수정 패스를 수행해 검토 재시도에 맞춘 일관성을 확보했다.

## Round 6 - Terminology
- 문서 앞부분에 `용어집(Glossary)`를 추가해 핵심 용어 기준을 한곳에서 먼저 확인할 수 있게 했다.
- `슬롯(queries)` 표기를 `slot/query`로 통일해 슬롯/쿼리 혼용 표현을 정리했다.
- Stage1/BCE 관련 챕터 제목에서 `objectness(logit/probability)` 표기를 맞춰, 로짓과 확률의 관계가 드러나도록 했다.
- 좌표계 표기를 `normalized/global -> feature coordinates`로 통일해 좌표 단위 전환 문맥을 명확히 했다.
- ROI 관련 챕터 제목을 `ROI/ROI Align` 표기로 통일해 영역(ROI)과 연산(ROI Align) 구분을 일관되게 맞췄다.
- 매칭 챕터 제목에 `matched/negative`를 명시해 양성/음성 슬롯 의미를 용어 기준과 연결했다.
- 목차와 본문 챕터 제목을 동일 용어로 동기화해 장 이동 시 용어 단절이 없도록 정리했다.

## Round 7 - Misconceptions & Quiz
- 1~20장 전체를 대상으로 `오해/바로잡기`와 `체크 퀴즈`를 전수 점검했고, 누락되어 있던 `오해/바로잡기` 섹션을 각 장에 추가했다.
- 최소 기준(10개)보다 크게 넘겨, 총 60개 문항(오해/바로잡기 20개 + 퀴즈 40개)을 장별 주제에 맞게 재작성했다.
- 기존의 추상/반복 질문(예: 공통 shape 질문)을 제거하고, 각 장 제목/본문에서 직접 답할 수 있는 문항으로 바꿔 함정형 문제를 없앴다.
- 퀴즈 난이도를 장 내에서 `개념 확인 1문항 -> 적용/진단 1문항` 구조로 맞춰 점진적 학습 흐름을 강화했다.
- 후반 장(14~20장)은 단순 암기 대신 손실 가중치, 매칭 한계, 디버깅 우선순위, 실험 설계 판단을 묻는 문항으로 상향 조정했다.

## Round 8 - Debugging
- 19장에 `실전 디버깅 순서(10분 루틴)`을 추가해 초심자가 바로 실행할 수 있는 점검 순서를 고정했다.
- 디버깅 로그 스니펫 2개를 추가해 `출력 범위 확인`과 `매칭/손실 분해 확인`을 바로 복붙해 실행할 수 있게 했다.
- stage별 진단을 `backbone -> body -> ROI -> head -> matching -> loss` 순으로 재구성하고, 각 단계마다 최소 3개 체크 포인트를 넣었다.
- 각 stage에 `실패 예시`를 1개씩 추가해 "어떤 증상일 때 무엇을 의심할지"를 사례 기반으로 제시했다.
- `if X happens, check Y` 표를 넣어 자주 발생하는 6개 증상에 대한 즉시 확인 항목을 한눈에 찾게 했다.
- loss 하강만으로 정상 학습이라 판단하는 실수를 막기 위해 `손실 항 기여도(가중치 x 평균손실)` 점검을 명시했다.
- matching 실패(`matched_ratio=0`)와 좌표계 불일치(`cxcywh/xyxy`)를 분리 진단하도록 체크 순서를 구체화했다.
- ROI 실패 케이스에서 자주 놓치는 `좌표 스케일`과 `batch index` 확인을 독립 항목으로 분리해 재현 가능한 점검 루틴을 만들었다.
